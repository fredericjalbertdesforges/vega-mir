# Bundled demonstration data

These three JSON files ship with the repository so the notebooks in
`notebooks/` are fully self-contained: no external download, no
authentication, no clone of the upstream Cygnus pipeline required.

## Files

| File | Size | Used by | Contents |
|---|---|---|---|
| `composer_counts.json` | ~32 KB | `02_paper_reproduction.ipynb` | Per-composer aggregated marginal + joint (bigram) counts on the 15-symbol Cygnus alphabet. Aggregated across **all available pieces** of each composer in the upstream Cygnus corpus (Bach: 145 pieces, Chopin: 198, Beethoven: 145, Glass: 11, Richter: 13, etc.). This is the canonical input for reproducing the paper findings. |
| `composer_pieces.json` | ~340 KB | reference / future notebooks | Five representative sample pieces per composer with raw scale-degree sequences. Not consumed by the current notebooks but ships as a reference for users who want to inspect real symbolic input or to write their own per-piece analyses. |
| `paper_reference.json` | ~40 KB | `02_paper_reproduction.ipynb` | Subset of published numerical values from the Cygnus arXiv paper artefacts (`reports/kl_analysis_20260417/`): Shannon `H` per composer, Zipf `(alpha, R²)` for marginal and transitions, and the asymmetric KL matrix restricted to the 8 bundled composers. Used for parity comparisons in notebook 2. |

## Bundled composers (8)

Selected to span the paper's flagship findings:

- 6 historical: Bach, Haydn, Beethoven, Chopin, Liszt, Rachmaninoff
- 2 neoclassical: Glass, Richter

The pairs Haydn-Beethoven and Liszt-Rachmaninoff are documented stylistic lineages that the KL matrix recovers without supervision.

## Provenance

All three files were generated by `notebooks/_data_extraction/extract_from_cygnus.py`, which runs against a local clone of the Cygnus pipeline (not bundled). The methodology mirrors `CYGNUS/scripts/shannon_zipf_v2.py`:

- per-piece scale-degree sequences with consecutive duplicates collapsed (`sans self-repetitions`)
- key-relative mapping via Cygnus's `_parse_key`, `_chord_root_semitone`, `_interval_to_degree`
- Jeffreys-Laplace smoothing (`alpha = 0.5`)
- log base 2 (bits)

Scale degrees use ASCII shorthand: `bII` for `flat-II`, `#IV` for `sharp-IV`, lowercase for modal alterations (`i`, `iv`, `v`).
